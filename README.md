# Employee Performance Tracker (Oracle APEX)
------
A complete HR management application built with **Oracle APEX**, designed to manage employees, departments, and performance metrics. This README provides a step‑by‑step guide to the project setup, schema design, and APEX configuration.
-------------------------------------------

## 📑 Project Overview
The application includes:
- **Employee Directory** with search and export features.
- **Departments Management** for adding and editing departments.
- **Performance Metrics** to define KPIs.
- **Employee Performance Entry** for recording scores.

-------------------------------------------
## 🔧 Import Instructions
1. Download `employee-performance-tracker.sql` from this repo.
2. In Oracle APEX → App Builder → Import.
3. Upload the file and follow the wizard.
4. The application will appear in your workspace with the same ID.
-------------------------------------------

## ⚙️ Step 1: Database Schema

### Users Table
```sql
CREATE TABLE users (
    user_id        NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    username       VARCHAR2(50) NOT NULL,
    email          VARCHAR2(100) UNIQUE NOT NULL,
    password       VARCHAR2(100) NOT NULL,
    role           VARCHAR2(20) CHECK (role IN ('ADMIN','VIEWER')),
    department_id  NUMBER,
    is_active      NUMBER(1) DEFAULT 1,
    CONSTRAINT fk_user_dept FOREIGN KEY (department_id) REFERENCES departments(department_id)
);
```

### Departments Table
```sql
CREATE TABLE departments (
    department_id   NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    department_name VARCHAR2(100) NOT NULL,
    location        VARCHAR2(100)
);
```

### Performance Metrics Table
```sql
CREATE TABLE performance_metrics (
    metric_id     NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    metric_name   VARCHAR2(100) NOT NULL,
    description   VARCHAR2(255)
);
```

### Employee Performance Table
```sql
CREATE TABLE employee_performance (
    perf_id        NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    user_id         NUMBER NOT NULL,
    metric_id      NUMBER NOT NULL,
    score          NUMBER(5,2),
    evaluation_date DATE DEFAULT SYSDATE,
    comments       VARCHAR2(255),
    CONSTRAINT fk_emp FOREIGN KEY (emp_id) REFERENCES employees(emp_id),
    CONSTRAINT fk_metric FOREIGN KEY (metric_id) REFERENCES performance_metrics(metric_id)
);
```

## ⚙️ Step 3: Oracle APEX Pages

### Employee Directory
- **Interactive Report** with search by name/email.
- **Export Options**: Excel, PDF, CSV.
- **Form Page** for adding/editing employees.
- **LOV** for Department selection.

### Departments Management
- **Interactive Report** for departments.
- **Form Page** for adding/editing departments.
- **Validation**: Department name required, unique.

### Performance Metrics
- **Interactive Report** for KPIs.
- **Form Page** for adding/editing metrics.
- **LOV** used in Employee Performance Entry.

### Employee Performance Entry
- **Form Page** with LOVs for Employee and Metric.
- **Score field** with validation (0–100).
- **Comments field** optional.
- **Redirect** after submit → Performance Reports page.

## 🚀 Step 5: Deployment
1. Import schema into Oracle Database.
2. Create an APEX application with Interactive Reports and Forms.
3. Apply CSS in **Shared Components → Cascading Style Sheets**.
4. Test with dummy data.
5. Deploy to production.
